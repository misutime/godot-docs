# Godot 英文文档本地开发和部署

这份文档面向 `godot-docs` 仓库本身，主要说明英文文档源码的本地开发和构建。

如果你的目标是以简体中文文档为主，请优先使用：

```text
D:\misutime\godot-docs-l10n\开发和部署.md
```

简中工作区应站在：

```text
D:\misutime\godot-docs-l10n
```

而不是单独站在 `D:\misutime\godot-docs`。

## 技术栈

这个仓库是 Godot 官方英文文档的源码仓库。

- 文档源码格式：reStructuredText，文件后缀主要是 `.rst`
- 文档构建工具：Sphinx
- HTML 主题：`sphinx_rtd_theme`
- 依赖来源：`requirements.txt`
- Sphinx 配置入口：`conf.py`
- 文档首页入口：`index.rst`
- 线上构建环境：Read the Docs 会指定自己的 Python 版本

本地开发不需要特意固定 Python 版本。你可以直接使用本机已有的 Python，例如 Python 3.13；只要依赖能正常安装、Sphinx 能正常构建即可。

## 和简中仓库的关系

`godot-docs` 只包含英文 `.rst` 源文档。简体中文翻译在另一个仓库：

```text
D:\misutime\godot-docs-l10n
```

当前推荐结构是：

```text
D:\misutime\godot-docs-l10n
  docs\                指向 D:\misutime\godot-docs 的 Junction
  sphinx\po\zh_CN\     简中翻译
  _build\zh_CN\        简中 HTML 构建产物
```

也就是说：

- 改英文源码：编辑 `D:\misutime\godot-docs`。
- 改简中翻译：编辑 `D:\misutime\godot-docs-l10n\sphinx\po\zh_CN`。
- 构建英文站：可以在 `D:\misutime\godot-docs` 执行。
- 构建简中站：应该在 `D:\misutime\godot-docs-l10n` 执行。

## 平时开发一定要先 build 吗

要预览最终网页效果，就需要经过 Sphinx 构建。

原因是浏览器不能直接把 `.rst` 源码当成 Godot 文档网站来显示。Sphinx 会把 `.rst`、主题、静态资源、自定义扩展等一起转换成 `_build/html` 里的静态 HTML。

日常开发有两种方式：

- 手动构建：改完文档后运行一次 build，再刷新浏览器。
- 自动构建：使用 watch 工具监听文件变化，保存后自动重新构建并刷新预览。

如果只是改文字、查源码、做批量搜索，不需要先 build。只有需要检查网页排版、链接、代码块、图片、目录结构等最终效果时，才需要构建预览。

## 使用 uv 初始化环境

进入仓库：

```powershell
cd D:\misutime\godot-docs
```

用本机 Python 创建虚拟环境并安装依赖：

```powershell
uv venv
uv pip install -r requirements.txt
```

如果想确认当前环境里的 Python 版本：

```powershell
uv run python --version
```

之后所有命令都可以通过 `uv run` 执行，不一定要手动激活 `.venv`。

## 手动构建和预览

构建英文 HTML：

```powershell
uv run sphinx-build -M html . _build
```

也可以使用仓库自带的 Windows 脚本：

```powershell
.\make.bat html
```

构建结果在：

```text
D:\misutime\godot-docs\_build\html
```

直接打开这个文件即可预览：

```text
D:\misutime\godot-docs\_build\html\index.html
```

如果希望用本地 HTTP 服务预览：

```powershell
uv run python -m http.server 8000 -d _build\html
```

然后访问：

```text
http://localhost:8000
```

## 自动构建预览

如果你会频繁改英文文档，推荐安装 `sphinx-autobuild`：

```powershell
uv pip install sphinx-autobuild
```

启动自动构建服务器：

```powershell
uv run sphinx-autobuild . _build\html --host 127.0.0.1 --port 8000
```

然后访问：

```text
http://127.0.0.1:8000
```

之后修改 `.rst`、图片或静态资源时，`sphinx-autobuild` 会自动重新构建。浏览器通常也会自动刷新。

## 常用开发命令

完整构建 HTML：

```powershell
uv run sphinx-build -M html . _build
```

清理构建产物：

```powershell
Remove-Item -LiteralPath .\_build -Recurse -Force
```

严格模式构建，把警告当作错误：

```powershell
uv run sphinx-build -W -b html . _build\html
```

只看 Sphinx 支持的构建目标：

```powershell
.\make.bat help
```

## 部署静态 HTML

英文文档最终产物是静态网站。部署英文站时只需要部署构建后的 `_build/html` 目录。

生产构建建议先清理旧产物：

```powershell
Remove-Item -LiteralPath .\_build -Recurse -Force
uv run sphinx-build -M html . _build
```

部署目录：

```text
D:\misutime\godot-docs\_build\html
```

可以部署到任意静态网站服务，例如：

- Nginx
- Apache
- GitHub Pages
- Cloudflare Pages
- Netlify
- Vercel 静态站点
- 任意对象存储静态网站托管

部署时站点根目录应指向 `_build/html`，而不是仓库根目录。

## 简中构建速查

简中构建请切换到 `godot-docs-l10n`：

```powershell
cd D:\misutime\godot-docs-l10n
```

第一次构建、语言配置变化、或发现页面仍是英文时，使用干净构建：

```powershell
Remove-Item -LiteralPath .\_build\zh_CN -Recurse -Force

$env:READTHEDOCS_LANGUAGE = "zh_CN"
$env:SPHINX_TAGS = "i18n"

uv run sphinx-build -E -b html -t i18n `
  -D locale_dirs="D:/misutime/godot-docs-l10n/sphinx/po" `
  docs _build\zh_CN
```

之后日常增量构建不要加 `-E`：

```powershell
$env:READTHEDOCS_LANGUAGE = "zh_CN"
$env:SPHINX_TAGS = "i18n"

uv run sphinx-build -b html -t i18n `
  -D locale_dirs="D:/misutime/godot-docs-l10n/sphinx/po" `
  docs _build\zh_CN
```

`-E` 会强制 Sphinx 丢弃 doctree 缓存并重读全部页面。Godot 文档的 class reference 很大，所以每次都带 `-E` 会非常慢。日常开发应保留 `_build\zh_CN\.doctrees` 缓存。

启动简中预览：

```powershell
uv run python -m http.server 8000 -d _build\zh_CN
```

访问：

```text
http://localhost:8000
```

## 常见问题

### 找不到 sphinx-build

说明依赖还没有安装到当前环境。重新执行：

```powershell
uv pip install -r requirements.txt
```

然后用：

```powershell
uv run sphinx-build --version
```

### python 命令启动的是 Windows Store 占位程序

可以优先使用 `uv` 管理 Python 环境，并确认 `uv` 选到的解释器：

```powershell
uv run python --version
```

如果 `uv venv` 没有选到你想用的 Python，可以显式指定本机版本，例如：

```powershell
uv venv --python 3.13
```

### 修改后页面没变化

英文构建时，先确认构建命令没有报错，然后刷新浏览器。如果仍然没变化，可以清理后重新构建：

```powershell
Remove-Item -LiteralPath .\_build -Recurse -Force
uv run sphinx-build -M html . _build
```

如果是简中页面仍然显示英文，通常是旧 doctree 缓存或 `locale_dirs` 没指到 `godot-docs-l10n\sphinx\po`。回到 `D:\misutime\godot-docs-l10n`，执行“简中构建速查”里的干净构建命令。

### 部署后样式或链接异常

确认部署的是 `_build/html` 目录里的内容，而不是整个源码仓库。`index.html` 应该位于部署站点根目录。
